# Recapitulação do HTML e CSS e teste do git no terminnal do VSCODE
| Comandos | Descrição |
| :--- | :---: |
| `git init` | Cria um novo repositório Git na pasta atual. |
| `git add <ficheiro>` | Adiciona um ficheiro à área de preparação (staging area). |
| `git add .` | Adiciona todas as alterações da pasta atual à área de preparação. |
| `git commit -m "mensagem"` | Guarda as alterações preparadas no histórico do projeto com uma mensagem descritiva. |
| `git log` | Mostra o histórico de commits. |
| `git status` | Mostra o estado atual do repositório, incluindo ficheiros modificados e preparados para commit. |
| `git diff` | Mostra as diferenças entre as alterações atuais e a última versão guardada. |
| `git branch` | Lista as branches locais e indica qual está ativa. |
| `git switch -c <nome-da-branch>` | Cria uma nova branch e muda para ela. |
| `git switch <nome-da-branch>` | Muda para uma branch existente. |
| `git checkout <nome-da-branch>` | Forma mais antiga de mudar de branch (continua a funcionar). |
| `git merge <nome-da-branch>` | Junta as alterações de outra branch na branch atual. |
| `git remote add <nome> <url>` | Adiciona um repositório remoto. |
| `git push <remoto> <branch>` | Envia os commits locais para o repositório remoto. |
| `git pull <remoto> <branch>` | Obtém as alterações do remoto e integra-as na branch atual. |
| `git fetch` | Obtém as alterações do repositório remoto sem as integrar automaticamente. |
